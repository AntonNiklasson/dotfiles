- In any written communication, be extremely concise. Sacrifice grammar for the sake of concision.

## Git 

- Do not commit code without my approval
- Always prefix new branches with `an/` as a namespace
- I prefer smaller, focused, commits
- Whenever I ask you to open a new pull request, always do it as a draft unless explicitly told otherwise
- When updating title and body of a pull request, make sure to not overwrite any changes I've made via GitHub web UI
